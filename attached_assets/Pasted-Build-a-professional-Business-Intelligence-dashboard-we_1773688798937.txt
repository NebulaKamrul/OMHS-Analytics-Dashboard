Build a professional Business Intelligence dashboard web application for a hospital operations analytics use case.

The goal is to simulate an internal analytics/reporting tool that hospital leadership could use to monitor operational performance. This project is intended for a technical internship portfolio, so it should prioritize realism, clarity, proper data modeling, and strong engineering practices rather than flashy UI or startup-style aesthetics.

This project should feel like an internal analytics tool used by analysts, operations managers, or BI teams in a healthcare organization.

-----------------------------------

PROJECT GOALS

This project should demonstrate:

• relational data modeling
• SQL-based analytics queries
• dashboard reporting
• data visualization
• filterable reports
• enterprise-grade UI discipline
• clear backend data flow
• structured architecture

-----------------------------------

-----------------------------------

STRICT DESIGN RULES

Avoid vibe-coded website patterns.

COLORS & VISUAL

• No default purple gradients unless brand-appropriate
• No sparkles or emojis in headings
• No glowing hover effects
• Use neutral enterprise palette
• Maintain consistent contrast

TYPOGRAPHY

• Define a consistent type scale
• Avoid oversized headings with ultra-thin body text
• Maintain consistent line height
• Maintain consistent paragraph spacing
• Enforce clear hierarchy

LAYOUT & COMPONENTS

• Consistent component placement across pages
• Define and reuse 2-3 border radius values maximum
• Subtle hover states only (2-4px lift max)
• Icons scaled proportionally to text
• Remove non-functional social icons
• Avoid decorative UI elements with no functional purpose

ANIMATIONS & INTERACTIONS

• Use easing curves (cubic-bezier)
• Stagger timing intentionally
• Animations must serve a UX purpose
• Avoid decorative animation

UX BEHAVIORS

• Loading states for all async actions
• Progress indicators for long operations
• Skeleton screens for data-heavy sections
• Functional filters and interactions
